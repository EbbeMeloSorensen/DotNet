How to make a migration after having changed the code for the domain model (using the code first approach)

1) Åbn Package Manager Console (som er et view i Visual Studio)
2) Sæt et relevant Default Project, f.eks. PR.Persistence.EntityFrameworkCore.PotgreSQL
3) Eksekver Add-Migration efterfulgt af et navn på pågældende migration, f.eks. InitialMigration
   (Hvis der ikke er en "Migrations" folder under det projekt, der blev anført som default, så laves der en)

Det forklares bl.a. af Mosh Hamedani i lektion 8 ("Demo: Code-First Workflow") af Udemy kurset "Entity Framework in Depth: The Complete Guide".
Jeg stødte et eller andet sted på at man passende kunne operere med at have et Console Application projekt, som afhang af det ønskede plugin
for at lave databasen (eller var det selve migreringen?). Jeg kan dog ikke lige huske hvor, men det har nok været et Udemy kursus.
